# Requirements: 多模态数据管理平台 (EmbedAI DataHub)

## 业务目标

为具身机器人研发团队提供一站式数据管理平台，覆盖从传感器原始数据入库、处理、标注到训练集导出的完整链路，以加速模型迭代。

## 核心角色

| 角色 | 类型 | 职责 |
|------|------|------|
| 机器人 / 仿真器 | 系统 | 产生并上传 MCAP/HDF5 数据 |
| 数据工程师 | 内部 | 管理流水线、数据质检、数据集策划 |
| 标注员 | 内部 + 外包 | 执行视频/图像/时序多模态联合标注 |
| 审核员 | 内部 | 审核标注质量 |
| 算法工程师 | 内部 | 搜索数据、发起导出、对接训练 |
| 管理员 | 内部 | 用户管理、权限配置、项目管理 |

## 数据特征

| 属性 | 说明 |
|------|------|
| 格式 | MCAP（主）、HDF5 |
| 典型文件大小 | 1–10 GB / 条 |
| 总规模 | TB 级 |
| MCAP 内容 | 多路 Topic（Camera、LiDAR、IMU、Gripper 等） |
| HDF5 内容 | 层级数据集（关节角、力矩、状态轨迹等） |

## 数据采集方式

- **批量上传**：机器人任务结束后通过 SDK/HTTP 上传 MCAP/HDF5 文件
- **实时流式接入**：机器人运行时实时写入平台（gRPC 流或 MQTT）

## 标注需求

- 视频/图像：2D 框、关键点、语义分割
- 时序/信号：片段标记（动作阶段、接触事件等）
- 多模态联合：同一时间窗口下跨 Camera/LiDAR/Force 的关联标注

## 训练对接

- 导出目标：云端对象存储（S3 兼容）
- 导出格式：原始文件 + 标注 JSON、HuggingFace datasets 格式（可选）
- 支持按 Dataset 版本、过滤条件增量导出

## 权限控制

- 基于角色（RBAC）：Admin / Engineer / Annotator（内部）/ Annotator（外包）
- 外包标注员仅可访问分配给自己的任务及对应数据片段
- 项目级数据隔离

## 关键约束

| 约束 | 描述 |
|------|------|
| 大文件上传 | 单文件最大 10 GB，需分块上传 + 断点续传 |
| 流式接入延迟 | 实时写入延迟 < 500ms |
| 标注数据一致性 | 标注与原始数据强绑定，版本化管理 |
| 存储成本 | 原始数据冷热分层，减少频繁访问成本 |

## 已确定架构决策

| 决策 | 内容 |
|------|------|
| 质量评分 | 帧率稳定性 + 传感器完整性 + 信号质量三维加权，< 0.3 自动隔离 |
| 流式断线 | seq_num 续传，断线 > 30s 生成独立 Episode，session_id 关联 |
| 标注工具 | MVP: Label Studio 自托管，REST API + Webhook 集成 |
| 任务分配 | 初期手动 + 负载展示，预留自动分配（skill_tags + Round-Robin） |
| 版本策略 | 引用快照，不复制文件，版本不可变 |
| 导出格式 | WebDataset (P0) + 裸文件/JSON (P0) + HF Parquet (P1) |

详见 [decisions.md](decisions.md)

## 成功标准

1. 数据从采集到可标注的端到端延迟 < 5 分钟
2. 标注任务从创建到分配 < 1 分钟
3. 数据集导出到云存储 < 30 分钟（100GB 以内）
4. 系统支持 10+ 并发标注员同时工作
